# Task - 작업지시서 자동 생성

작업 내용을 분석하여 자동으로 작업지시서를 생성하는 스킬입니다.

## 사용법

```
/task [작업 설명]
```

## 예시

```
/task UserDb에 IsDeptObjectiveWriter 필드 추가
/task MemberDb 테이블 제거 및 v_MemberListDB 재작성
```

## 동작

1. 작업 내용 분석
2. 작업 규모 판단 (simple/detailed)
3. 적절한 템플릿 선택 (TEMPLATE_simple.md 또는 TEMPLATE_detailed.md)
4. works/tasks/ 폴더에 작업지시서 생성 (날짜_순번_작업명.md 형식)
5. 관련 파일 경로, 변경 전/후 코드, 테스트 계획 포함

## 작업지시서 생성 규칙

### 파일명 형식
- `YYYYMMDD_XX_작업명.md`
- 예: `20260114_01_add_dept_objective_writer.md`

### 작업 규모 판단 기준

**간단한 작업 (TEMPLATE_simple.md)**:
- 1-2개 파일 수정
- 50줄 미만 변경
- 단순 필드 추가/삭제
- 간단한 뷰 수정

**복잡한 작업 (TEMPLATE_detailed.md)**:
- 3개 이상 파일 수정
- 테이블 구조 대폭 변경
- 외래키 관계 수정
- 새 기능 추가
- 프로시저/함수 생성

## 작업지시서 포함 내용

1. **작업 배경**: 왜 이 작업이 필요한가?
2. **작업 내용**: 무엇을 어떻게 수정하는가?
3. **변경 전/후**: 코드 비교
4. **영향도 분석**: 어떤 파일들이 영향을 받는가?
5. **테스트 계획**: 어떻게 검증할 것인가?
6. **체크리스트**: 작업 단계별 확인 사항

## 자동 처리 사항

- 최근 작업지시서 번호 자동 증가
- 템플릿 자동 선택
- 관련 파일 자동 탐색
- 테스트 쿼리 자동 생성 (DB 작업 시)
