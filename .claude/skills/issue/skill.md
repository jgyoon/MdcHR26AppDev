# Issue - 이슈 생성 및 관리

프로젝트 이슈를 생성하고 관리하는 스킬입니다.

## 사용법

```
/issue [이슈 제목 또는 설명]
/issue update [이슈번호] [상태]
/issue list
```

## 예시

```
/issue UserDb 비밀번호 보안 강화 필요
/issue update 011 완료
/issue list
```

## 동작

### 1. 새 이슈 생성 (`/issue [제목]`)

1. **issue.md 읽기**
   - 최근 이슈 번호 확인
   - 현재 진행 중인 이슈 파악

2. **새 이슈 파일 생성**
   - works/issues/XXX_영문제목.md 생성
   - 템플릿 기반 이슈 파일 작성

3. **issue.md 업데이트**
   - 이슈 목록에 추가
   - 현황 통계 업데이트

### 2. 이슈 상태 업데이트 (`/issue update`)

- 이슈 파일의 상태 변경 (진행중/완료/보류)
- issue.md의 현황 통계 자동 업데이트
- 완료 이슈는 "최근 완료된 이슈" 목록으로 이동

### 3. 이슈 목록 조회 (`/issue list`)

- 진행 중인 이슈 목록
- 최근 완료된 이슈 (최근 5개)
- 이슈 현황 통계

## 이슈 파일 구조

### works/issues/XXX_title.md
```markdown
# 이슈 #XXX: [제목]

**날짜**: YYYY-MM-DD
**상태**: 진행중 / 완료 / 보류
**우선순위**: 높음 / 중간 / 낮음
**관련 이슈**: [#YYY](YYY_title.md)

---

## 개발자 요청
...

## 해결 방안
...

## 진행 사항
- [ ] 작업지시서 작성
- [ ] 코드 작성
- [ ] 테스트

---

## 관련 문서
**작업지시서**:
- [작업지시서](../tasks/YYYYMMDD_XX_title.md)
```

## 자동 처리 사항

### 이슈 번호 관리
- 마지막 이슈 번호 자동 증가
- 3자리 형식 (001, 002, ..., 012)

### 영문 제목 생성
- 한글 제목을 영문으로 변환
- 소문자와 언더스코어 사용
- 예: "비밀번호 보안 강화" → "password_security_enhancement"

### 관련 이슈 링크
- 상대 경로로 자동 링크
- 작업지시서 연결

### 통계 자동 계산
- 총 이슈 개수
- 완료/진행/보류 개수
- 진행률 계산

## 이슈 관리 규칙

1. **새 이슈 생성 시**
   - issue.md 확인 → 최근 번호 파악
   - 개별 파일 생성
   - issue.md 업데이트

2. **이슈 상태 변경 시**
   - 개별 파일 상태 업데이트
   - issue.md 통계 업데이트

3. **이슈 완료 시**
   - 테스트 결과 기록
   - 개발자 피드백 섹션 작성
   - "최근 완료된 이슈" 목록으로 이동

## 연속 작업 지원

다음 세션에서 작업 시:
1. issue.md 자동 확인
2. 진행 중인 이슈 파악
3. 관련 이슈 링크 따라가기
4. 컨텍스트 자동 복원
