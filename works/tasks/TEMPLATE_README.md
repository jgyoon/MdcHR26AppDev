**최종 업데이트**: 2025-11-13

---

## 📋 템플릿 종류

### 1. TEMPLATE_simple.md (간소화 템플릿)
**용도**: 간단하고 빠른 작업
**작성 시간**: 약 3-5분
**추천 사용 케이스**:
- ✅ 단순 버그 수정 (1-2개 파일, 소량 코드 변경)
- ✅ UI 스타일 조정 (CSS 변경)
- ✅ 텍스트/문구 수정
- ✅ 간단한 로직 수정 (조건문 하나, 변수 하나 등)
- ✅ 설정 값 변경

### 2. TEMPLATE_detailed.md (상세 템플릿)
**용도**: 복잡하고 중요한 작업
**작성 시간**: 약 10-20분
**추천 사용 케이스**:
- ✅ 새로운 기능 추가
- ✅ 복잡한 버그 수정 (여러 파일, 많은 코드 변경)
- ✅ 아키텍처 변경
- ✅ API 변경
- ✅ 데이터베이스 스키마 변경
- ✅ 보안 관련 작업
- ✅ 성능 최적화

## 📝 템플릿 사용 절차

### Step 1: 템플릿 복사
```bash
# 간소화 템플릿
cp works/tasks/TEMPLATE_simple.md works/tasks/YYYYMMDD_XX_task_name.md

# 상세 템플릿
cp works/tasks/TEMPLATE_detailed.md works/tasks/YYYYMMDD_XX_task_name.md
```

### Step 2: 파일명 규칙
```
YYYYMMDD_XX_task_name.md

예시:
20251113_01_button_color_fix.md        (간소화)
20251113_02_bulk_message_feature.md    (상세)
```

**규칙**:
- `YYYYMMDD`: 날짜 (8자리)
- `XX`: 순번 (01, 02, 03, ...)
- `task_name`: 영문 소문자 + 언더스코어
- 간결하고 명확하게 (3-5단어)

### Step 3: 템플릿 항목 채우기

#### 간소화 템플릿 필수 항목
1. ✅ 작업명
2. ✅ 날짜, 파일, 작업 유형
3. ✅ 작업 개요 (목적, 배경)
4. ✅ 수정 내용 (코드 before/after)
5. ✅ 테스트 항목 (최소 1개)
6. ✅ 완료 조건

#### 상세 템플릿 필수 항목
1. ✅ 모든 간소화 템플릿 항목
2. ✅ 문제 상황 (증상, 테스트 결과)
3. ✅ 원인 분석 (정상/문제 케이스 비교)
4. ✅ 수정 로직 설명
5. ✅ 다양한 테스트 시나리오 (3개 이상)
6. ✅ 예상 결과 (수정 전/후 비교)
7. ✅ 주의사항

**선택 항목**: 로그 분석, 백업 안내, 관련 코드 참조, 롤백 계획

### Step 4: 개발자 검토
- 작성된 작업지시서를 개발자에게 공유
- 승인 받은 후 작업 진행

## 🎯 작성 팁

### 간소화 템플릿
1. **간결하게**: 핵심만 작성 (1-2페이지)
2. **코드 중심**: 설명보다는 코드 before/after에 집중
3. **테스트 명확히**: 개발자가 5분 내에 테스트 가능하도록

### 상세 템플릿
1. **맥락 제공**: 왜 이 작업이 필요한지 배경 설명
2. **원인 분석**: 문제의 근본 원인 파악
3. **완전성**: 나중에 이 문서만 보고도 이해할 수 있도록
4. **재현 가능**: 테스트 시나리오를 따라하면 반드시 재현되도록

### 공통 팁
- ✅ 코드는 실제 라인 번호 명시
- ✅ 파일 경로는 프로젝트 루트 기준 상대 경로
- ✅ 스크린샷이 필요하면 별도로 첨부
- ✅ 관련 이슈 링크는 반드시 포함
- ✅ 테스트 결과는 ✅ 또는 ❌로 명확히 표시

## 🔄 템플릿 업그레이드

### 간소화 → 상세로 전환해야 할 때
작업 중 다음 상황이 발생하면 상세 템플릿으로 전환:

1. **복잡도 증가**: 생각보다 코드 변경이 많아짐
2. **영향 범위 확대**: 다른 기능에 영향을 주는 것을 발견
3. **추가 버그 발견**: 수정 중 관련 버그를 추가로 발견
4. **테스트 실패**: 예상치 못한 테스트 실패 발생

**전환 방법**:
1. 기존 작업지시서를 `*_simple_backup.md`로 백업
2. 상세 템플릿으로 새로 작성
3. 기존 내용을 복사하여 확장

## ❓ FAQ

### Q1: 템플릿을 반드시 다 채워야 하나요?
**A**: 아니요. 필수 항목만 채우고, 해당 없는 항목은 삭제하세요.

### Q2: 작업 도중 템플릿을 바꿔도 되나요?
**A**: 네. 복잡도가 증가하면 간소화→상세로 전환 가능합니다.

### Q3: 매우 간단한 작업도 작업지시서가 필요한가요?
**A**: 네. 간소화 템플릿으로 3분이면 작성 가능하며, 추후 추적에 도움됩니다.

### Q4: 긴급 작업은 어떻게 하나요?
**A**: 간소화 템플릿으로 빠르게 작성 후 작업하고, 나중에 상세 내용 보완 가능합니다.

### Q5: 여러 작업을 한 파일에 작성해도 되나요?
**A**: 권장하지 않습니다. 작업별로 파일을 분리하여 추적성을 높이세요.
